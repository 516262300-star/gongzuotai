# 拼多多周报指标定义

本文件用于固定周报指标口径，避免脚本、Notion 周报和人工理解不一致。

## 核心指标

| 指标 | 口径 | 备注 |
| --- | --- | --- |
| 销售额 | 周期内销售额合计 | 待确认是否扣除退款 |
| 订单数 | 周期内订单数合计 | 待确认是否排除取消订单 |
| 访客数 | 周期内访客数合计 | 如果数据源没有访客数，则暂不计算转化率 |
| 转化率 | 订单数 / 访客数 | 访客数为 0 时不计算 |
| 退款金额 | 周期内退款金额合计 | 待确认是否按退款发生日统计 |
| 退款率 | 退款金额 / 销售额 | 销售额为 0 时不计算 |
| 推广花费 | 周期内广告消耗合计 | 后续可由广告同步项目提供 |
| 推广 ROI | 销售额 / 推广花费 | 推广花费为 0 时不计算 |

## 维度

- 周期
- 店铺
- 商品
- SKU
- 类目，待补充
- 活动，待补充

## 需要你确认

- 拼多多后台导出文件的真实列名，当前脚本使用 `date,shop_name,product_name,sku,visitors,orders,revenue,refund_amount,ad_spend`。
- 销售额是否使用付款金额、成交金额还是结算金额。
- 是否需要扣退款后销售额。
- 周报周期使用自然周还是自定义周期。
